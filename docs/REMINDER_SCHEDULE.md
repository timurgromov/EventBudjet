# Reminder Schedule

Текущий production flow reminder-сообщений для пользователей Telegram-бота:

1. `reminder_d2`
   - отправляется после `2` дней неактивности

2. `reminder_d7`
   - отправляется после `7` дней неактивности

3. `reminder_followup`
   - первый раз отправляется после `14` дней неактивности
   - дальше повторяется каждые `7` дней, пока пользователь не вернётся в Mini App или не заблокирует бота

## Важные правила

- Любой новый визит пользователя в Mini App сбрасывает таймер неактивности.
- Порядок стадий не откатывается назад: после прохождения `d2` и `d7` пользователь остаётся в weekly follow-up стадии.
- Повторные follow-up сообщения ротируются по нескольким универсальным шаблонам, чтобы не отправлять один и тот же текст бесконечно.

## Обратная совместимость

- Исторические записи с кодом `reminder_d14` продолжают учитываться как уже отправленные follow-up сообщения.
- Для новых отправок используется код `reminder_followup`.
