# DEPLOY_WORKFLOW.md

Постоянная схема работы для проекта `EventBudjet`.

## Цель

Не смешивать:

- локальную разработку;
- прод на VPS;
- Telegram bot polling;
- временные туннели.

## Источник истины

- Код хранится в git/GitHub.
- Локальная машина нужна для разработки и проверки изменений.
- VPS нужен для рабочего прод-запуска.

## Постоянные правила

1. Код меняется локально.
2. Локально бот не запускается.
3. Локально для проверки поднимаются только:

```bash
docker compose up -d frontend backend postgres nginx
```

4. Telegram bot работает только на VPS.
5. Перед прод-обновлением изменения коммитятся в git и пушатся в GitHub.
6. На VPS всегда деплоится код из репозитория, а не ручные несохраненные правки.

## Постоянные команды

Локально поднять проект без `bot`:

```bash
npm run local:up
```

Локально остановить проект:

```bash
npm run local:down
```

Задеплоить на VPS:

```bash
npm run deploy:vps
```

Посмотреть основные логи VPS:

```bash
npm run logs:vps
```

## Правильный порядок работы

1. Внести изменения локально.
2. Проверить локально без `bot`.
3. Закоммитить изменения:

```bash
git add .
git commit -m "Your message"
git push
```

4. Выполнить:

```bash
npm run deploy:vps
```

5. Проверить:

```bash
npm run logs:vps
```

## Что нельзя делать

- Не запускать локально `bot`, если продовый бот уже работает на VPS.
- Не использовать старые локальные туннели как постоянную прод-схему.
- Не править прод вручную в обход git, если это не аварийный фикс.
- Не держать разные compose-структуры локально и на VPS.

## Когда допустим аварийный фикс на VPS

Только если прод уже сломан и нужен срочный recovery.

После этого обязательно:

1. перенести изменения обратно в локальный репозиторий;
2. закоммитить их;
3. вернуть обычный workflow через git.

## Если меняется токен или доступ

- Обновить `.env` на VPS.
- Перезапустить нужный контейнер:

```bash
sudo docker compose up -d bot
sudo docker compose up -d backend
```

- Если меняется Telegram token, перезапускать и `bot`, и `backend`.

## Быстрый смысл

- Локально: разработка.
- GitHub: источник кода.
- VPS: рабочий прод.
